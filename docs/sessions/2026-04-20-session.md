## Non-obvious findings (for future reference)

1. **`grep -c '.' \|\| echo "0"` is broken**: on no-match, `grep -c` prints "0" AND exits 1, so `||` fires and appends another "0" → result "0\n0". Breaks `[ -gt 0 ]` under bash 5.x. Use `wc -l | tr -d ' '`.
2. **`stat -f` is BSD-only**: GNU stat treats `-f` as filesystem-info mode and pollutes stdout (fs info for the target file) before falling through. Always try `-c` (GNU) first, `-f` (BSD) second, each captured into a local before emitting.
3. **Grep patterns starting with `-` need `--`**: even single-quoted, grep parses `-` as an option → exit 2 with `grep: invalid option -- ' '`. Use `grep -qE -- 'PATTERN'`.
4. **`pr-merged` linked-issue regex misses cross-repo `owner/repo#N`** — closure-guard must route to TBT itself, not rely on `pr-merged`. Widening the regex is a safe follow-up.
5. **The approval gate had silently been broken the whole time** — it was never exercised before because merging PRs without acceptance criteria on issues (or without the `Closes #N` path) didn't hit it. Our `/project-board verify` + careful closure workflow exposed the latent bug.
